# Инструкция по работе с системой Qwen Image Edit

## Что это такое

Система автоматической обработки фотографий на базе **Qwen Image Edit 2509** через ComfyUI.  
Позволяет работать с **двумя изображениями одновременно**: менять фон, переносить одежду, добавлять объекты, объединять персонажей — с сохранением лица и идентичности человека.

---

## Принцип работы: как комбинировать два фото

Система использует технику **ImageStitch** — два фото склеиваются горизонтально в одно (слева направо), и модель получает единое изображение с контекстом обоих источников.

```
[Фото 1] + [Фото 2]  →  [Фото1 | Фото2]  →  Qwen видит оба → генерирует результат
```

**Правило расположения:** Фото 1 — основной объект (человек), Фото 2 — референс (одежда, фон, поза и т.д.)

---

## Что можно делать

### 1. Добавление/замена фона
**Фото 1:** человек на нейтральном фоне  
**Фото 2:** желаемый фон  
**Промпт:** `"она стоит на фоне из второго изображения. Оставь лицо, позу и одежду без изменений. Сохрани то же освещение"`

---

### 2. Смена одежды / образа
**Фото 1:** человек в текущей одежде  
**Фото 2:** референс одежды  
**Промпт:** `"на ней надет именно такой же наряд, как на втором изображении. Лицо, причёска и поза остаются прежними"`

> Можно одновременно сменить позу, фон и выражение лица — просто опиши это в промпте.

---

### 3. Добавление объекта в сцену
**Фото 1:** человек в сцене  
**Фото 2:** объект (сумка, аксессуар, предмет)  
**Промпт:** `"она держит сумку из второго изображения. Студийное освещение с мягкими тенями. Сумка видна полностью, с детальной проработкой материалов"`

---

### 4. Два персонажа / взаимодействие
**Фото 1:** первый персонаж  
**Фото 2:** второй персонаж  
**Промпт:** `"они стоят рядом, обнимаются, смотрят в камеру. Студийное освещение, плавный переход к фону на закате. Сохрани внешность обоих персонажей"`

---

### 5. Единый фон для серии фото
**Фото 1:** человек (разные позы)  
**Фото 2:** фон  
**Промпт:** `"она стоит, подняв руку к волосам, на фоне из второго изображения. Фон оставить без изменений. Сохранить то же освещение"`

---

## Как писать промпты

### Структура хорошего промпта

```
[Описание действия/позы] + [Что взять из фото 2] + [Что сохранить из фото 1] + [Фон/освещение]
```

### Примеры из tutorial

**Добавление объекта:**
> "она смотрит в камеру, слегка наклонившись, в динамичной позе. Одной рукой она напряженно играет мышцами, спортивная сумка зажата между ног. Студийное освещение с естественными мягкими тенями, фон — интерьер спортзала с оборудованием и едва заметными бликами, спортивная сумка видна полностью, с детальной проработкой материалов и текстур"

**Замена фона + сохранение позы:**
> "она стоит, запустив руку в волосы, на фоне изображения. Оставьте фон без изменений, но добавьте над ней надпись «VOGUE». Сохраните то же освещение"

**Смена одежды + новая поза:**
> "она удивлена, у нее открыт рот, на ней точно такой же комплект нижнего белья, как на картинке. Вид сбоку, крупный план, белая стильная комната, сидит на стуле, руки в волосах"

**Два персонажа:**
> "они стоят близко друг к другу, обнимаются, смотрят прямо в камеру и естественно улыбаются. Студийное освещение с мягкими естественными тенями. Плавный переход к пляжу на закате, золотистые солнечные лучи отражаются в песке и воде"

### Ключевые фразы для сохранения идентичности

| Что сохранить | Формулировка в промпте |
|---|---|
| Лицо | `"keep the exact same face, eyes, hairstyle"` |
| Поза | `"same pose, same perspective"` |
| Фон | `"leave the background unchanged"` |
| Освещение | `"same lighting, same shadows"` |
| Одежда | `"same clothing as the original"` |

### Negative prompt (что исключить)
```
blurry, distorted, artifacts, bad anatomy, deformed, low quality
```
Для замены одежды добавь название старой одежды:
```
pink shirt, polka dots, original clothing, blurry, distorted
```

## Запуск сервера

```bash
cd '/Users/gosha/Desktop/демо генератор/ComfyUI'
source venv/bin/activate
python main.py --force-fp32 --preview-method auto
```

> **ВАЖНО:** флаг `--force-fp32` обязателен. Без него на MPS (Apple GPU) модель выдаёт чёрные изображения (NaN).

Сервер запустится по адресу **http://127.0.0.1:8188**  
Интерфейс доступен в браузере по тому же адресу.

## Скрипты

### `bg_replace.py` — замена фона

Помещает ребёнка с одной фотографии на новый фон.

**Подготовка:**
1. Положи фото ребёнка в `input/`
2. Положи фото фона в `input/`

**Запуск:**
```bash
python3 bg_replace.py --child DSC08443.JPG --bg фон_1.jpg
python3 bg_replace.py --child DSC08443.JPG --bg фон_1.jpg --seed 123   # фиксированный seed
python3 bg_replace.py --all   # обработать все фото из input/
```

**Результат:** сохраняется в `output/`

---

### `test_girl_outfit.py` — перенос одежды

Надевает одежду с референсного фото на человека.

**Подготовка:**
1. Положи фото человека в `input/` (сейчас: `DSC08443.JPG`)
2. Положи фото одежды в `input/` (сейчас: `одежда.png`)

**Запуск:**
```bash
python3 test_girl_outfit.py
```

Чтобы сменить фото — отредактируй переменные в начале файла:
```python
IMAGE1 = "твоё_фото.JPG"   # человек
IMAGE2 = "одежда.png"       # референс одежды
```

**Промпт** описывает что именно надеть — редактируй переменную `PROMPT` в файле.

---

### `test_tutorial.py` — универсальный шаблон мульти-редактирования

Принимает два изображения и текстовый промпт — делает любое редактирование.

**Подготовка:**
1. Положи оба изображения в `input/`

**Запуск:**
```bash
python3 test_tutorial.py
```

Отредактируй переменные в начале файла:
```python
IMAGE1 = "исходник.png"
IMAGE2 = "референс.png"
PROMPT = "твоё описание что нужно сделать"
NEGATIVE = "что не должно быть в результате"
```

## Как писать промпты

Промпт описывает **желаемый результат** (не инструкцию). Примеры:

- **Замена фона:** `"девочка стоит на улице в парке, зелёные деревья, солнечный день, фотостудийное качество"`
- **Одежда:** `"на ней надета красная куртка и белые кроссовки из второй картинки, лицо и поза не изменились"`
- **Поза:** `"она сидит на стуле, вид сбоку, руки в волосах, то же лицо"`

**Negative prompt** — что исключить: `"blurry, distorted, artifacts, bad anatomy, deformed"`

## Типичные проблемы

| Проблема | Причина | Решение |
|---|---|---|
| Чёрное изображение | Запущено без `--force-fp32` | Перезапусти сервер с флагом |
| `Connection refused` | Сервер не запущен | Запусти `python main.py --force-fp32` |
| Лицо изменилось | Слабый промпт | Добавь в промпт: `"keep the exact same face, eyes, hairstyle"` |
| Медленная генерация | Норма для MPS | 4 шага занимают ~2-5 минут на M4 |
